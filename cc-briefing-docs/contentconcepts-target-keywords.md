## Instructions

Each topic cluster has:
- **Pillar Keyword** — main, high-volume keyword (competitive; target with a comprehensive service or guide page)
- **Cluster Keywords** — 5–10 related subtopics (target with dedicated service pages or blog posts)
- **Long-Tail Keywords** — specific, lower-competition phrases (target with FAQs, blog articles, landing page copy)
- **Search Intent** — what the user wants: Informational (I), Commercial (C), Transactional (T)
- **Suggested Page** — the ContentConcepts page best positioned to rank for this keyword

## Usage Guidelines

### Writing New Content
1. Identify which cluster the topic belongs to
2. Target the cluster or long-tail keyword (not just the pillar — those are for service pages)
3. Reference and link to the pillar service page from blog posts
4. Include LSI keywords naturally throughout
5. Add internal links between related cluster content

### Optimizing Existing Pages
1. Confirm each service page targets its designated pillar keyword in: `<title>`, `<h1>`, first 100 words, URL slug, meta description
2. Check for keyword cannibalization between service pages
3. Ensure each page has a unique keyword focus (e.g., `/manuscript-editing-service` vs `/paper-editing-service` must be clearly differentiated)
4. Add FAQ sections using long-tail and PAA keywords
5. Update meta descriptions to include target keyword + key differentiator (24-hr turnaround, free certificate, $0.02/word)

### Content Calendar Planning
1. Use the 12-month theme calendar from the content strategy document
2. Map each monthly theme to 2–3 keywords from these clusters
3. Prioritize high-priority clusters first (Clusters 1, 2, 3, 4, 7)
4. Build out long-tail blog content to support each service page cluster
5. Track keyword rankings monthly in Ahrefs Rank Tracker
